# Contributing to VisionQwen

Contributions are welcome! VisionQwen is an independent fork and all pull requests are reviewed and merged directly here.

## Pull Requests

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the project builds and runs correctly on your target platform.
5. Make sure your code lints.
6. Open a pull request with a clear description of what changed and why.

## Issues

Use GitHub Issues to report bugs or request features. Please include:

- A clear description of the problem or request
- Steps to reproduce (for bugs)
- Expected vs actual behavior
- Platform (iOS / Android) and OS version

**Security issues:** Please report privately via [GitHub Security Advisories](https://github.com/tr10ne/visionqwen/security/advisories/new) rather than opening a public issue.

## Coding Guidelines

- iOS: follow Swift API Design Guidelines; use `async/await` where possible
- Android: follow Kotlin coding conventions; prefer coroutines over callbacks
- Keep changes focused -- one concern per PR
